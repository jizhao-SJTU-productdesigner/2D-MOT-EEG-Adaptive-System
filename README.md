**2D-MOT Neural Adaptive Closed-Loop Training System**

This repository contains the core source code of the 2D-MOT neural adaptive closed-loop intervention system used for empirical research in a thesis. Combining neuroergonomics and human-computer interaction (HCI) design principles, the system dynamically adjusts the physical difficulty of the tracking task by real-time monitoring of the subject’s EEG Theta/Alpha power ratio (TAR) and behavioral accuracy (ACC), aiming to keep the user in the optimal cognitive state of "flow" or "focused attention".



**I. Core Features**

\- Real-time closed-loop control: Uses a Python backend for millisecond-level EEG signal processing and achieves parameter synchronization with the Unity 3D frontend via UDP protocol.  

\- Six-state cognitive model: Built-in decision-making model covering \*underload boredom (Boredom)\*, \*mastery (Mastery)\*, \*flow (Flow)\*, \*unstable (Unstable)\*, \*overload struggling (Struggling)\*, and \*overload withdrawal (Withdrawal)\*, implementing a nonlinear difficulty adjustment strategy.  

\- Individualized baseline-driven: Automatically extracts EEG features from the subject’s eyes-closed resting state (EC) to establish a dynamic physiological arousal threshold ($Threshold = 2 \\times TAR\_{EC}$).  

\- Rigorous data alignment: Sends event markers via asynchronous UDP to ensure physiological signal segmentation is strictly synchronized with game phases (memory, tracking, selection, settlement).  

\- Multi-dimensional data logging: Automatically generates detailed experimental logs with physiological metrics (TAR median, standard deviation, etc.) and behavioral metrics (ACC, reaction time, peak speed) for subsequent academic analysis.



**II. System Architecture**

The system consists of two main parts:

1\. Python Backend (EEG Signal Processing Engine)

File: neuro\_mot\_processor.py  

\- Data acquisition: Receives raw EEG data from devices like OpenBCI via the LSL (Lab Streaming Layer) protocol.  

\- Signal preprocessing: Implements real-time filtering (Notch 50Hz, Bandpass 1-40Hz) and a MAD-z robust artifact rejection pipeline.  

\- Feature extraction: Calculates the power ratio (TAR) of Theta (4-7Hz) and Alpha (8-13Hz) in real time for occipital sites (O1, O2).

2\. Unity Frontend (Interaction and Decision-Making Module)

Core scripts:  

\- NeuroMotGame.cs: Overall system control hub, executing the "six-state model" adaptive algorithm and level state switching.  

\- MotTrialManager.cs: Manages the physical flow of a single trial (ball spawning, target assignment, motion control).  

\- TarReceiverUdp.cs: Asynchronously listens to and parses TAR feature packets from the backend, ensuring smooth UI rendering.  

\- SessionLogger.cs: Handles full-cycle experimental data logging and CSV export.  

\- MotBall.cs: Defines motion/collision logic and visual feedback for balls in a 2D physical environment.



**III. Core Algorithm Logic**

The system implements difficulty adaptation through the following (partially simplified) logic:

State---Trigger Condition---Intervention Strategy

Boredom---ACC ≥ 99% ∩ TAR ≤ Threshold---Significantly increase difficulty: +0.3 speed, add distractor and target balls

Mastery---ACC ≥ 99% ∩ TAR > Threshold---Smoothly increase difficulty: If at 4-ball limit, increase speed; if pushing to 5 balls, first decrease speed

Flow---70% ≤ ACC < 99% ∩ TAR > Threshold---Maintain state: Slightly increase speed by 0.1

Struggling---ACC < 70% ∩ TAR > Threshold---Precisely decrease difficulty: Keep 4-ball baseline but significantly reduce speed to relieve pressure

Withdrawal---ACC < 70% ∩ TAR ≤ Threshold---Rescue intervention: Significantly reduce speed, total balls, and target balls



**IV. Key Data Description**

The core data structure generated by this study is in All\_Sessions\_Summary.csv, including:  

\- Avg\_Accuracy: Full-cycle average accuracy.  

\- Peak\_Speed: Peak tracking speed achieved by the subject under adaptive intervention.  

\- Avg\_TAR: Average EEG arousal level during training.  

\- Baseline\_EC: Individual baseline for EEG state determination.





**2D-MOT 神经自适应闭环训练系统 (Neural Adaptive System)**

本仓库包含了用于学位论文实证研究的 2D-MOT 神经自适应闭环干预系统 的核心源码。该系统结合了神经工效学与人机交互（HCI）设计原则，通过实时监测受试者的脑电 Theta/Alpha 功率比（TAR）与行为正确率（ACC），动态调节追踪任务的物理难度，旨在将用户维持在“心流”或“专注意通”的最佳认知状态。



**一、核心特性**

\- 实时闭环控制：利用 Python 后端进行毫秒级 EEG 信号处理，并通过 UDP 协议与 Unity 3D 前端实现参数同步。

\- 六状态认知模型：系统内置了涵盖“欠载无聊 (Boredom)”、“精通 (Mastery)”、“心流 (Flow)”、“不稳定 (Unstable)”、“过载挣扎 (Struggling)”以及“过载撤离 (Withdrawal)”的决策模型，实现非线性的难度调控策略。

\- 个体化基线驱动：自动提取受试者闭眼静息态（EC）脑电特征，建立动态生理唤醒阈值（$Threshold = 2 \\times TAR\_{EC}$）。

\- 严谨的数据对齐：通过异步 UDP 发送事件标记（Markers），确保生理信号切分与游戏阶段（记忆、追踪、选择、结算）严格同步。

\- 多维数据落盘：系统自动生成包含生理指标（TAR 中位数、标准差等）与行为指标（ACC、反应时、峰值速度）的详细实验日志，支持后续学术分析。



**二、系统架构**

系统由两个主要部分组成：

1\. Python 后端 (EEG 信号处理引擎)

文件：neuro\_mot\_processor.py

\- 数据接入：通过 LSL (Lab Streaming Layer) 协议接收来自 OpenBCI 等设备的原始 EEG 数据。

\- 信号预处理：实现实时滤波（Notch 50Hz, Bandpass 1-40Hz）与 MAD-z 稳健伪迹剔除管线。

\- 特征提取：针对枕区位点（O1, O2）实时计算 Theta (4-7Hz) 与 Alpha (8-13Hz) 的功率比值 (TAR)。

2\. Unity 前端 (交互与决策模块)

核心脚本：

\- NeuroMotGame.cs：全系统控制中枢，负责执行“六状态模型”自适应算法及关卡状态切换。

\- MotTrialManager.cs：管理单个 Trial 的物理流程（小球生成、分配目标、运动控制）。

\- TarReceiverUdp.cs：异步监听并解析后台发来的 TAR 特征包，确保 UI 渲染不卡顿。

\- SessionLogger.cs：负责实验全周期的数据记录与 CSV 导出。

\- MotBall.cs：定义 2D 物理环境中小球的运动碰撞逻辑与视觉反馈。



**三、核心算法逻辑**

系统通过以下逻辑（部分简化）实现难度自适应：

状态判定---触发条件---干预策略

Boredom---ACC≥99%∩TAR≤Threshold---大幅提难：速度 +0.3，增加干扰球与目标球

Mastery---ACC≥99%∩TAR>Threshold---平滑提难：若在 4 球上限则提速；若冲击 5 球则先降速

Flow---70%≤ACC<99%∩TAR>Threshold---维持状态：微幅提速 0.1

Struggling---ACC<70%∩TAR>Threshold---精准降难：保持 4 球基准但大幅降速，缓解压力

Withdrawal---ACC<70%∩TAR≤Threshold---挽救干预：大幅降低速度、总球数与目标数



**四、关键数据说明**

本研究所产生的核心数据结构见 All\_Sessions\_Summary.csv，包含：

\- Avg\_Accuracy：全周期平均正确率。

\- Peak\_Speed：受试者在自适应干预下达到的峰值追踪速度。

\- Avg\_TAR：训练期间的平均脑电唤醒水平。

\- Baseline\_EC：个体脑电状态判定基准值。
